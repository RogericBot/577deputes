Méthodologie & RGPD

Détection bots : un visiteur est classé « bot » si son User-Agent matche un pattern connu (Googlebot, Bingbot, Claude, GPT, Wget, Curl, etc.) OU s'il ne contient pas la chaîne Mozilla (heuristique : tous les navigateurs réels la mettent ; son absence indique un script ou outil). UA vide = bot suspect.

Top User-Agents : seules des catégories agrégées sont stockées (ex : « Chrome desktop », « Bot — Googlebot »). Aucun User-Agent brut, aucune IP, aucun fingerprint n'est conservé.

Top pages : les UID sont remplacés par [uid] pour agréger (ex : /deputes/[uid] au lieu de /deputes/PA12345). Pas de query string conservée.

Visiteurs uniques : aucun cookie. Pour chaque requête on calcule sha256(sel-du-jour + IP + User-Agent) tronqué ; ce hash sert seulement à dédoublonner les visiteurs dans la journée, change chaque jour (donc impossible de suivre quelqu'un d'un jour à l'autre), et est purgé après ~2 jours. L'IP n'est jamais écrite — elle ne sert qu'au calcul du hash.

Provenance : seul le domaine du referrer est conservé (jamais l'URL complète, qui pourrait être sensible), normalisé en source lisible (« Recherche : Google », « Réseau : Discord », « (accès direct) »…). Recherches : les requêtes de la barre de recherche du site sont comptées en agrégé (minuscules, tronquées à 80 car.), jamais liées à un visiteur ni horodatées individuellement.

Rétention : agrégats journaliers conservés ~25 mois (limite CNIL), hashs éphémères ~2 jours ; purge automatique au changement de jour. Données stockées : data/analytics.db, jamais transmis à un tiers, jamais exporté, accessible uniquement depuis cette page (HTTP Basic Auth).
